3.5 Rancangan REST API
    *** SCREENSHOT/FOTO: image5.jpg ***
    (Diagram endpoint API menunjukkan base URL dan seluruh endpoint)

    Endpoint yang tersedia:
    1. Users API
       - GET    /api/users           (ambil semua user)
       - GET    /api/users/{id}      (ambil user by ID)
       - POST   /api/users           (tambah user)
       - PUT    /api/users/{id}      (update user)
       - DELETE /api/users/{id}      (hapus user)

    2. Donor API
       - GET    /api/donor           (ambil semua donor)
       - GET    /api/donor/{id}      (ambil donor by ID)
       - POST   /api/donor           (tambah donor)
       - PUT    /api/donor/{id}      (update donor)
       - DELETE /api/donor/{id}      (hapus donor)

    3. Donation API
       - GET    /api/donation              (ambil semua donasi)
       - GET    /api/donation/{id}         (detail donasi)
       - POST   /api/donation              (tambah donasi)
       - GET    /api/donation/donor/{id}   (riwayat donasi per donor)

    4. Request Darah API
       - GET    /api/request         (ambil semua permohonan)
       - GET    /api/request/{id}    (detail request)
       - POST   /api/request         (buat permohonan)
       - PUT    /api/request/{id}    (update status)
       - DELETE /api/request/{id}    (hapus request)

    5. Location API
       - GET    /api/location        (ambil semua lokasi)
       - GET    /api/location/{id}   (detail lokasi)
       - POST   /api/location        (tambah lokasi)
       - PUT    /api/location/{id}   (update lokasi)
       - DELETE /api/location/{id}   (hapus lokasi)

    6. Stock Darah API
       - GET    /api/stock           (ambil semua stok)
       - GET    /api/stock/{id}      (detail stok)
       - PUT    /api/stock/{id}      (update stok)

    7. Complaint API
       - GET    /api/complaint       (ambil semua keluhan)
       - POST   /api/complaint       (tambah keluhan)
       - DELETE /api/complaint/{id}  (hapus keluhan)

    8. Service History API
       - GET    /api/history                (ambil riwayat)
       - GET    /api/history/{id}           (detail riwayat)
       - POST   /api/history                (tambah riwayat)
       - GET    /api/history/request/{id}   (riwayat per request)
